### 新增文件

| 文件 | 行数 | 内容 |
|------|------|------|
| `js/wave-manager.js` | 454 | 波次状态机 (idle→wave_start→in_wave→wave_cleared→upgrade_select→boss→victory)、10波配置数据、敌人生成调度 |

| 文件 | 改动 |
|------|------|
| `js/game.js` | +443行: `updateWaveBattle()` 主循环、`playerTakeDamage()` 包装函数、`update()` 模式分流、技能栏UI初始化、波次模式的开始/重启/结束逻辑 |

### 架构设计

- **共存架构**: `update()` 中 `selectedMode === 'wave_battle'` 提前分流，新旧模式互不干扰
- **数据驱动**: 英雄/技能/波次配置均为纯数据对象，行为通过注册表映射
- **最大复用**: 复用现有 `dealDamageToEnemy`、`handleEnemyKill`、`playerHit`、`showUpgradePanel`、`spawnBoss`、`render()`、粒子/相机/血条/经验系统

## 2026-03-27 - 踢击特效升级 (Lv1~Lv5 视觉分级)

使用 `images/effects/踢击/` 下的 14 张现有素材，为踢击技能 Lv1~Lv5 设计了完全不同的视觉身份。

### 素材角色映射

| 角色 | 素材文件 | 说明 |
|------|---------|------|
| 基础前挥 | 踢击_5, _6 | 小角度金色扫击，轻量感 |
| 强化前挥 | 踢击_1, _2 | 大金色横向爆发，压迫感 |
| 前冲残痕 | 踢击_3, _12+ | 斜向线性冲击/贯穿轨迹 |
| 旋风踢主圈 | 踢击_7, _8 | 金色旋涡环 (侧视) |
| 旋风踢副圈 | 踢击_11 | 干净金色圆环 (俯视) |
| 强化旋风圈 | 踢击_9 | 火焰翼旋涡 (最强环) |
| 命中爆点 | 踢击_10 | 星芒放射 |
| 地面冲击圈 | 踢击_4 | 俯视地面震波环 |
| 强命中爆点 | 踢击_12 | 暗金爆裂 |
| 高级主踢 | 踢击_攻击瞬间前冲 | 带弧线旋涡的前冲图 |

### 各等级视觉设计

**Lv1 基础踢击** — 轻、短、单次、干净
- 施放: 踢击_5 (基础前挥)，小尺寸，8帧
- 命中: 踢击_10 (小星芒)，很小
- 无旋转圈、无地面冲击、无前冲线

**Lv2 强化踢击** — 更重、更亮、更远
- 施放: 踢击_2 (强化前挥)，更大更亮，11帧 + 踢击_3 (前冲残痕，低透明度)
- 命中: 踢击_10 (中星芒)，比Lv1更大
- 增加了前冲残痕但很克制

**Lv3 旋风踢** — 环绕、360°、形态变化
- 施放: 踢击_7 (旋涡环，持续旋转) + 踢击_11 (干净环，反向慢旋转，半透明)
- 命中: 踢击_10 (弱化星芒，避免遮挡旋风环)
- 彻底抛弃前方挥击逻辑，一眼看出是周身扫击

**Lv4 连环踢** — 双段、连击、节奏感
- 施放第一段: 踢击_6 (基础前挥)，立即出现
- 施放第二段: 踢击_1 (强化前挥)，延迟5帧，更大更亮
- 命中第一段: 踢击_10 (小星芒)
- 命中第二段: 踢击_12 (暗金爆裂，延迟5帧)，视觉上明显有两次打击

**Lv5 觉醒踢击** — 终结、层次最多、最强
- 施放层1: 踢击_攻击瞬间前冲 (高级弧线主踢)，最大尺寸
- 施放层2: 踢击_3 + 踢击_12+ (双贯穿冲击线)
- 施放层3: 踢击_9 (火焰翼旋涡外环，持续旋转)
- 施放层4: 踢击_4 (地面冲击圈，反向慢旋转)
- 命中: 踢击_10 (大星芒) + 踢击_12 (暗金爆裂) + 踢击_4 (命中点地面冲击环)

## 2026-03-27 - 代码模块化拆分

将 `index.html` 中约 8,800 行的单体 `<script>` 块拆分为 10 个独立 JS 文件，提升可维护性和可读性。

### 拆分结构

| 文件 | 行数 | 内容 |
|------|------|------|
| `js/config.js` | 250 | 全局游戏配置 (GAME_CONFIG)、流派配置、割草模式、对象池系统、空间分区、经验值系统 |
| `js/weapons.js` | 920 | 武器定义 (踢击/匕首/闪电/火焰)、武器系统函数、燃烧/火雨机制、自动攻击系统、武器进化系统 |
| `js/relics.js` | 638 | 遗物状态追踪、基础/史诗/传奇遗物定义、宝箱系统 (4级分层)、割草升级面板、经验条 UI |
| `js/save.js` | 290 | 存档系统 (本地存储)、Supabase 排行榜、货币显示 |
| `js/ui.js` | 211 | 玩家血条系统、存活时间计算、成就奖励领取、局内结算奖励计算 |
| `js/audio.js` | 396 | Canvas/上下文初始化、音效系统、背景音乐 (BGM)、受击视觉反馈 (红屏) |
| `js/player.js` | 217 | 角色图片加载、跑步/死亡动画帧、狂战之心动画、待机动画系统 |
| `js/map.js` | 130 | 地图主题、世界尺寸与摄像机系统、地图碰撞系统、安全位置生成 |
| `js/shop.js` | 1207 | 游戏状态变量、升级池、波次系统、成就系统、局外商店、主页背景动画、天赋系统、遗物栏 UI |
| `js/game.js` | 4523 | 玩家状态、触屏/键盘控制、菜单交互、教程/世界观、游戏流程、技能系统、敌人生成/Boss AI、主循环 (update/render)、结算/胜利、游戏重置 |

### 加载顺序

```
config → weapons → relics → save → ui → audio → player → map → shop → game
```

所有文件共享全局作用域，按依赖关系顺序加载，无需模块打包工具。

### 未改动内容

- HTML 结构和 CSS 样式保留在 `index.html` 中
- 游戏逻辑和功能无任何变更
- 外部依赖 (Supabase SDK) 加载方式不变
